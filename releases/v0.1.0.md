# v0.1.0 — Initial Release

**Released:** 2026-03-28

## Highlights

pico-server-auth is an embeddable auth server module for pico-boot applications. It provides JWT issuance, wallet-based authentication, and a JWKS endpoint that integrates seamlessly with pico-client-auth.

## Features

- **Wallet authentication** — Challenge/verify flow supporting three signature algorithms:
    - ML-DSA-65 (post-quantum, NIST FIPS 204)
    - Ed25519
    - secp256k1
- **JWT issuance** — RS256 access tokens and refresh tokens with configurable expiry.
- **JWKS endpoint** — `/auth/jwks` serves the public key set so pico-client-auth can validate tokens in-process or remotely.
- **Password login** — Admin bootstrap login via `/auth/login` (disabled by default).
- **Challenge store** — `ChallengeStore` protocol with TTL-based `InMemoryChallengeStore`. Replaceable with Redis or database-backed implementations.
- **Embedded mode** — Runs inside any pico-boot application alongside your own controllers.
- **Auto-discovery** — Registered via `pico_boot.modules` entry point for zero-config setup.

## Configuration

All settings are under the `server_auth` prefix:

| Setting | Default |
|---|---|
| `issuer` | `http://localhost:8100` |
| `audience` | `pico` |
| `algorithm` | `RS256` |
| `access_token_expire_minutes` | `15` |
| `refresh_token_expire_days` | `7` |
| `challenge_ttl_seconds` | `60` |
| `auto_create_admin` | `false` |

## Dependencies

- pico-ioc >= 2.2.0
- pico-fastapi
- pico-client-auth (for `@allow_anonymous`)
- cryptography
- python-jose
